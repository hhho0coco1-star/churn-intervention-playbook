# 이탈 대응 플레이북 생성기 (채널톡 AX 컨설턴트용)

## 이 스킬이 하는 일
커머스 이탈 상황을 받아, **공개 출처만 근거로** "어떤 이탈 세그먼트에 어떤 개입을 할지"를 판단하는 플레이북을 만든다.
핵심 테제: **이탈 대응의 기본값은 쿠폰이 아니라 '왜 떠났는지 먼저 아는 것'이다.**

## 입력
- (필수) `references/public-sources.md` — 유일한 근거 원천.
- (선택) 컨설턴트 시나리오: 업종/객단가대/관찰 이탈지점/세그먼트 힌트 — 익명·공개안전 정보만, 사적 고객데이터 금지.
- (기본) 입력 없으면 붙박이 합성 데모로 실행하고, 리포트 상단에 "⚠️ 합성 예시 데이터"를 명시한다.

## 절대 원칙
1. 공개·검증가능 출처(references)만. 추측·사적경험·출처없는 숫자 금지.
2. 모든 판정에 출처 ID(B1/C2/V1/P1…) 부착.
3. 배포 기능(C*)과 제품 방향(V*)을 구분 표기.
4. 날짜 명시 + 12개월↑ 출처 ⏳(재확인 권고). P*는 이론 근거라 신선도 예외.
5. 로그 편집·발췌·삭제 금지.

## 원인-우선 3게이트 (핵심 판단)
각 이탈 세그먼트를 아래 게이트에 순서대로 통과시킨다.

- **G1. 원인을 아는가?** 모르면 → 🎙️ **전화 인터뷰 먼저** (V1 채널톡 방향). 쿠폰 아님.
- **G2. 쿠폰으로 고쳐지는 문제인가?** 제품·UX·배송 원인(B2)이면 → 🛠️ **제품팀 피드백.**
- **G3. 경제성이 되는가?** 마진 잠식·체리피커·bad-fit이면 → ⛔ **개입 안 함.** 근거는 초록 범위 원리만(P1·P2): 쿠폰 사용자는 비사용자보다 가격탄력적, 체리피커는 품목당 더 저렴하게·체계적으로 사는 합리적 세그먼트. 초록에 없는 "딜성향/가격민감 상단" 표현·특정 효과크기 주장 금지.
- 세 게이트를 모두 통과한 세그먼트에만 🎫 선별 쿠폰을 권고하고 우선순위를 매긴다.

### G3 가드레일 (필수 병기)
모든 🎫 쿠폰 권고에는 출처ID(P1·P2)와 함께 "잠식 리스크는 원리상 존재하나 실제 크기는 이 도구가 단정하지 않는다 — 고객사 holdout A/B로 확인 권고"를 반드시 병기한다.

### G3 손익분기 임계치
```
python scripts/breakeven.py <V> <m> <d>
```
- d ≥ m 이면 자동 ⛔ 후보(할인이 마진 이상).
- 아니면 T = d/(m−d) = "쿠폰 상환한 '원래 살 사람' 대비 증분 구매자가 최소 T배 나와야 본전". 라우팅 표에 반드시 표기한다(린터 강제).

## 출력 리포트 (8섹션)
1. 이탈 원인 진단(사실) 2. 개입 라우팅 표(핵심 판단) 3. ⛔개입안함이 정답인 세그먼트 4. 우선순위(🎫만) 5. 🎬영상반사대비 6. ⚠️확인필요 7. ⏳신선도 8. 출처목록

정확한 표 형식은 `examples/sample-playbook.md`를 그대로 따른다.

## 자기검증
```
python scripts/check_playbook.py <리포트.md>
python scripts/breakeven.py <V> <m> <d>
```
출처 생존·신선도 + 판단 무결성(출처실존·논리모순·가드레일완비)을 검사해 위반 시 FAIL(exit 1). 이 스크립트는 구조·정합성만 보며 경제성 판단의 진위는 검증하지 않는다.

## 완료 기준
- 모든 판정에 출처 ID가 붙어 있다. 이탈(사실)과 개입 권고(판단)가 구분된다. 배포/제품방향이 출처로 구분된다. 개입안함 세그먼트가 별도 명시된다. 확인불가 항목이 정직하게 표기된다.
