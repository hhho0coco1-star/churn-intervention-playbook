# 예선 질문 5문항 답변 — churn-intervention-playbook (채널톡)

> 각 문항 글자 수 상한(공백 포함): Q1 800 / Q2 800 / Q3 1000 / Q4 800 / Q5 800.
> Q2 출처 URL: https://baymard.com/lists/cart-abandonment-rate

## 1. 무엇을, 누가, 어떤 상황에서 쓰나요?

한 문장: 커머스 고객사의 이탈 상황을 넣으면 "어떤 이탈 세그먼트에 전화 인터뷰·선별 쿠폰·제품팀 피드백·개입 안 함 중 무엇을 할지"를 공개 출처 근거로 판단해 플레이북을 생성하는 Codex 플러그인입니다.

누가·어디서: 채널톡(채널코퍼레이션)의 AX 컨설턴트가 여러 커머스 고객사를 반복 진단할 때 씁니다. 이탈이 보이면 현장의 기본 반응은 "쿠폰부터 뿌리자"인데 바로 여기서 막힙니다 — 원인을 모른 채 쿠폰을 쓰면 원인이 배송·UX·제품일 때 못 고치고, 원래 살 사람이나 할인만 좇는 고객에게까지 마진이 새어 나갑니다.

## 2. 왜 이 문제를 선택했나요?

이탈은 커머스의 가장 크고 보편적인 문제입니다 — 평균 장바구니 이탈률 70.22%(Baymard, 50개 연구 평균, 무료 공개). 채널톡도 이 문제를 AI로 정면 겨냥합니다. 이 플러그인의 테제는 "이탈 대응의 기본값은 쿠폰이 아니라 왜 떠났는지 먼저 아는 것"입니다.

## 3. 플러그인은 어떻게 작동하나요?

이탈 세그먼트를 원인-우선 3게이트(G1 원인→G2 쿠폰적합성→G3 경제성)에 순서대로 통과시켜 8섹션 플레이북을 만듭니다. 모든 판정은 references/public-sources.md의 공개 출처(B/C/V/P)로만 근거하며 출처 ID를 붙입니다. check_playbook.py 린터가 규율을 강제합니다.

## 4. AI를 어떻게 썼나요?

AI에게 맡긴 것: 출처 실검색·verbatim 확보, 코드(린터·손익분기 계산기)를 TDD로 작성, 문서 초안·정리. 사람이 직접 판단한 것: 페르소나·핵심 판단축(3게이트)·"이탈≠쿠폰" 테제, 출처 채택/폐기 기준. 받아들이지 않은 AI 제안: 폐기한 Baymard "48%" 수치와 "incrementality 20/80" 수치, "체리피커=가격민감 상단 꼬리" 표현 — 실제 검증 가능한 출처가 없거나 유료 본문에만 있어 폐기했습니다.

## 5. 어떻게 검증했나요?

입력→결과 예시: "가격민감·양(+)마진" 세그먼트 입력 → 3게이트 모두 통과 → 🎫 선별 쿠폰 + 손익분기 T=0.5 + holdout 병기. 정상 리포트 2종은 린터 PASS(exit 0), 적대적 픽스처 8종은 FAIL(exit 1) 확인 — 날조 출처·G3실패인데 🎫·holdout 누락 등. 단위 테스트 17개 통과. 아직 부족한 것: Codex 런타임 end-to-end 구동은 심사자가 처음 수행합니다.
